# Initial cluster deployment using site.yml

Once you have configured your environment, run the playbook site.yml to perform the initial cluster deployment:

cd ~/Openshift-on-SimpliVity
# ansible-playbook -i hosts site.yml --vault-password-file .vault_pass

site.yml is a wrapper for a number of playblooks that perform different functions:

ansible-playbook -i hosts playbooks/prepare.yml
ansible-playbook -i hosts playbooks/provision.yml
ansible-playbook -i hosts playbooks/configure.yml
ansible-playbook -i hosts playbooks/poweron.yml
ansible-playbook -i hosts playbooks/finish.yml

# Prepare

The preparation stage includes the following tasks:

  • Install client tools
  • Create infrastructure and cluster folders
  • Populate /etc/hosts
  • Configure and start up HTTP server on Ansible box
  • Set up firewall rules

# Provision

The provision stage includes the following tasks:

  • Prepare Ignition data files
  • Provision bootsrap, master and CoreOS worker nodes
  • Provision and power on infrastructure and supporting nodes, including load balancers and NFS

# Configure

The configuration phase consists of:

  • Configuring DRS rules
  • Configuring infrastructure nodes - Configure DNS, DHCP. Set up disks, RedHat subscription.
  • Configuring supporting nodes - Configure load balancer and NFS nodes. Set up disks, RedHat subscription.

# Power on

This stage powers on any nodes that have not already been started. It waits for the OpenShift API server port on the master nodes to be ready and checks that access to port 22 is available.

# Finish

A number of tasks are performed at the end to ensure that the cluster has been successfully deployed:

  • Wait until the OpenShift installer reports that bootstrap is complete
  • Approve any pending CSRs
  • Wait for the image registry to be ready and configure persistent volumes for the NFS server hosting the registry
  • Verify that all the cluster operators are available
  • Wait for the installation to complete
