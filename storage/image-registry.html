# Image registry

The default vSphere Cloud Provider does not support the ReadWriteMany access mode required by the Image Registry. As a result, the playbook for initial cluster deployment, site.yml deploys an NFS virtual machine. A number of NFS shares are exported and the Image Registry uses one of these.

The number of shares that the playbooks creates can be customized using the variable num_nfs_shares in the configuration file group_vars/all/vars.yml. Only one share is required by the Image Registry service.
